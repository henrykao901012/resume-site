💡 專案經歷

不規則影像拼接系統開發

專題佳作獲獎 | 與兩位組員共同完成 | 指導教授：蔡宇軒

🏆 榮獲校內專題競賽「佳作」殊榮

研究動機：
現有影像拼接工具多仰賴固定形狀或順序，對於不規則影像的處理效果不佳。專題旨在開發一套可處理不規則形狀拼接的演算法，提升使用彈性與品質。

技術核心：

  • 使用 Python + OpenCV + NetworkX 開發拼接系統
  • SURF (Speeded-Up Robust Features) 特徵點抽取與匹配，高效處理拼接影像
  • 建立 weighted graph 並應用 Maximum Spanning Tree (MST) 找出最佳拼接順序
  • 結合圖論與特徵相似度分析，完成多張圖片自動拼接

技術實作流程：

  1. Graph Construction：使用 networkx 創建圖論模型，每個節點代表一張影像，利用 SURF 特徵匹配計算節點間的相似度權重
  2. Maximum Spanning Tree Computation：使用 MST 概念識別最相關和重疊的連接，確保影像重建的最佳順序

成果特色：

  • 可處理多種影像場景：同地不同時間、多人合照等不規則拼接需求
  • 運用 PicoCollage、Photo & Video Collage Maker 等工具驗證，本系統顯著減少影像扭曲
  • 成功處理不同場景（合併、合照、同地點不同時間）的拼接任務

金融數據K線圖視覺化系統

獨立開發 | 2024

🏆 實現百萬級數據點的流暢視覺化，支援年度到分鐘級別無縫縮放
高效能 大數據處理 響應式設計 效能優化

專案動機：
起於對量化交易的興趣，需要一套自己的看盤系統，現有金融圖表工具在處理大量歷史數據時常有效能瓶頸，無法提供流暢的互動體驗， 如TradimgView 且需要會員。本專案旨在開發一套高效能的K線圖視覺化系統，解決大數據量下的渲染效能問題。

核心技術實作：

  • 效能優化：實作二分查找快速定位資料，節流控制確保 30fps 流暢體驗
  • 智能採樣：實作 LTTB (Largest Triangle Three Buckets) 演算法，保留視覺重要特徵同時減少渲染點數
  • 動態顯示模式：根據數據密度自動切換 K線圖/線圖模式，確保任何縮放級別都能清晰呈現
  • 自適應軸標：時間軸根據跨度自動調整格式（年/月/日/時/分），價格軸動態適應可見範圍

  • 支援從年度視圖到分鐘級別的 10,000x 縮放範圍

高效能浮點數 Radix Sort 演算法實作與優化

獨立開發 | 2024

🏆 實現多種 Radix Sort 變體，最佳版本效能超越 std::sort 達 5.5 倍

專案動機：
研究如何將傳統用於整數排序的 Radix Sort 演算法擴展至浮點數排序，並透過多種優化策略提升效能，挑戰 C++ STL 的標準排序演算法。

技術實作：

  • 開發 6 種不同的 Radix Sort 變體，包含 LSD、MSD 及多執行緒版本
  • 實作 IEEE 754 浮點數轉換技術，巧妙處理正負數排序問題
  • 採用混合排序策略：小資料集使用 Insertion Sort，大資料集使用 Radix Sort
  • 實作多執行緒並行化，有效利用多核心處理器

效能成果（1 億筆雙精度浮點數）：

演算法版本 執行時間（秒） 相對於 std::sort
MSD Radix Sort (多執行緒) 8.79 5.32x 更快
MSD Radix Sort (遞迴) 9.85 4.75x 更快
Radix Sort (16-bit buckets) 17.96 2.60x 更快
std::sort (基準) 46.71 1.00x

技術亮點：

  • 創新的浮點數位元操作技術，實現高效的基數排序
  • 記憶體存取模式優化，提升 CPU 快取命中率
  • 實作完整的正確性驗證，確保與標準排序結果一致
